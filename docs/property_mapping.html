10. Priority & Override Rules

When multiple sources provide the same field, the following precedence applies:

PrioritySourceContext
1 (highest)Intune ExportFor displayName, publisher, version, appDescription — IE wins if _intuneExportImported flag is set AND field was already populated
2PSADT ScriptFor displayName, publisher, version — script values populate first via importPsadtState()
3Project FilesEdit Existing mode — spa-wizard-state.json snapshot takes highest priority (lossless round-trip), falling back to individual file parsing
4 (lowest)INITIAL_STATEWizard defaults (e.g., deployMode: 'Silent', restartBehavior: 'basedOnReturnCode')
⚠️ IMPORTANT deployMode defaults to Silent across all modes (new and refactored). For refactored titles, if the Intune export contains -DeployMode X in the install command line, that value is carried over. PSADT 4.1 automatically defaults to Auto at runtime when the flag is omitted from the command line, so there is no need to explicitly set it.
ℹ️ NOTE AppProcessesToClose is always shown in the builder's variable section, even when empty. It is the single source of truth — there is no separate "Close Apps" field in the Deploy Behavior tab.
