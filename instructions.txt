This project is about automating application packaging and deployment to Intune and JAMF depending on the target OS.  
Itune for Windowes software titles and JAMF for MAC OS software titles.

I have the following folder structure:

old_template contains my initial pass at this project (it only covers windows software titles).  I want you to use it as a reference to 
understand what I want to achieve. Here I created a template for Win32 application using PSADT.  
I have a pinned version of PSADT and scripts to support execution of the pipeline.  I have tested the pipeline and it works. 

The initial approach I took is to create a generic template that can be used to create each software title and then create a script to 
automate the packaging and deployment of the application to Intune and JAMF using gitlab pipeline. 
The issue with that initial approach was that each software title repo would have had the full copy of the repo and with 2000 titles, 
that would have been waste.   

The other folders represent my attemps to seperate the model for both platforms such that I can reference reusabel and commont components and 
end up with a clean structure like the followin:

Sotware Title repo:

- Only app-specific content and metadata:
- source installer
- app-specific scripts
- metadata for Windows and/or Mac
- deployment inputs, but not generic deployment engine code

For both platforms:

1. Build layer
Creates the installable artifact.

2. Deployment layer
Publishes and assigns that artifact to the management platform.

So the corrected target model becomes:

- Windows build framework: PSADT
- Windows deployment automation: Intune deployment modules
- macOS build framework: macOS packaging framework
- macOS deployment automation: Terraform/Jamf modules
- shared CI templates: orchestration glue
- title repos: app-specific content only

That is the target model I want to create which gives me:
- one repo per software title
- platform-specific folders inside title repos
- Windows build framework externalized
- macOS build framework externalized
- Windows deployment externalized
- Jamf deployment externalized
- CI templates externalized


The new repo structure contains placeholders for the new model and for what I think the content shoudl be.  I want you to use it as a reference to 
understand what I want to achieve.  

What each repo should publish or expose
- packaging-standards Publishes: schemas, examples, standards docs
- psadt-enterprise Publishes: immutable versioned framework bundle
- macos-packaging-framework Publishes: immutable versioned framework bundle
- intune-deployment-modules Publishes: reusable deployment scripts/modules
- terraform-jamf-modules Publishes: reusable Terraform modules
- gitlab-ci-templates Publishes: reusable GitLab includes
- titles/* Publishes: software-specific packages and metadata

Then I want to further develop and complete the code to support both target platforms make it more modular and reusable.  

I want to use gitlab pipeline to automate the packaging and deployment of the application to Intune and JAMF using gitlab pipeline. 
I want to use PSADT for windows and a similar framework for mac.  I want to use Terraform for Jamf deployment.  
I want to use Graph for intune publishing and assignment.  
I want to use a single pipeline for both platforms.  The pipeline should be triggered by a tag.  The tag should be in the format of vX.Y.Z.  
I want to document the code and the pipeline.
